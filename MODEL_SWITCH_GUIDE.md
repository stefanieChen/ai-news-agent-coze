## 🔄 切换模型方法

### 方法1: 修改配置文件（推荐）

编辑 `config/agent_llm_config.json` 文件，修改 `active_model` 字段：

```json
{
    "active_model": "balanced",  // 修改为想要的模型key
    ...
}
```

**示例**：
```json
{
    "active_model": "cost-effective"  // 切换到成本最低模型
}
```

### 方法2: 使用环境变量（临时切换）

设置环境变量 `COZE_ACTIVE_MODEL` 来临时切换模型：

```bash
export COZE_ACTIVE_MODEL=pro
```

或者在运行时指定：

```bash
COZE_ACTIVE_MODEL=pro python src/main.py
```

**注意**: 环境变量的优先级高于配置文件。

## 💡 使用建议

### 根据场景选择：

1. **预算有限** → 使用 `cost-effective`
2. **日常使用** → 使用 `balanced`（默认）
3. **追求质量** → 使用 `pro`
4. **复杂分析** → 使用 `thinking`
5. **长文本处理** → 使用 `kimi`
6. **稳定性优先** → 使用 `legacy`

### 根据频率选择：

- **高频使用（每天多次）** → 推荐 `cost-effective`
- **中频使用（每天一次）** → 推荐 `balanced`
- **低频使用（每周一次）** → 可以使用 `pro`

## 🔍 查看当前使用的模型

运行Agent时，会在控制台输出当前使用的模型信息：


## ⚠️ 注意事项

1. **切换模型后会立即生效**，无需重启服务
2. **不同模型的费用不同**，建议根据预算选择
3. **环境变量优先级高于配置文件**
4. **如果指定的模型不存在**，会自动使用 `balanced` 模型并给出警告

## 🆘 常见问题

**Q: 如何知道当前使用的是哪个模型？**
A: 查看控制台输出，会显示模型名称和ID。

**Q: 切换模型后会影响之前的对话记录吗？**
A: 不会，对话记录独立于模型选择。

**Q: 可以同时使用多个模型吗？**
A: 不可以，一次只能使用一个模型。如需切换，修改配置即可。

**Q: 哪个模型最省钱？**
A: `cost-effective` 模型成本最低。

**Q: 哪个模型质量最好？**
A: `pro` 和 `kimi` 模型质量最高，但成本也更高。

**Q: legacy 模型是什么？**
A: legacy 是豆包1.6旧版本模型，作为稳定性和兼容性备选方案。

**Q: active_model 和 legacy 有什么区别？**
A: `active_model` 是一个配置项，指定当前激活使用哪个模型；`legacy` 是一个模型key，代表豆包1.6这个特定模型。
